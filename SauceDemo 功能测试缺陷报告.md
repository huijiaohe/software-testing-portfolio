## 缺陷详情

### BUG-001：用户名前后空格未自动trim
- **严重程度**：一般  
- **模块**：表单校验  
- **复现步骤**：
  1. 打开登录页 https://www.saucedemo.com/
  2. 用户名输入 ` standard_user `（前后各加一个空格）
  3. 密码输入 `secret_sauce`
  4. 点击 Login
- **实际结果**：登录失败，显示错误提示“Username and password do not match”
- **预期结果**：系统应自动去除前后空格，登录成功
- **建议修复**：在登录请求前对用户名进行 trim() 处理

---

### BUG-003：Tab键焦点高亮缺失
- **严重程度**：中等  
- **模块**：可用性/无障碍  
- **复现步骤**：
  1. 登录后进入商品列表页
  2. 连续按 Tab 键移动焦点
- **实际结果**：焦点顺序正确（筛选框 → 商品按钮 → 图片链接），但无任何高亮样式，用户无法定位当前焦点位置
- **预期结果**：焦点所在元素应有明显的高亮边框（如 outline 或背景色变化）
- **建议修复**：为所有可聚焦元素添加 `:focus` 样式，例如 `outline: 2px solid #e74c3c`

---

### BUG-004：结账信息返回后未保留已填内容
- **严重程度**：一般  
- **模块**：结账流程  
- **复现步骤**：
  1. 登录后添加商品到购物车
  2. 进入购物车，点击 Checkout
  3. 在结账信息页面填写 First Name、Last Name、Zip Code
  4. 点击 Cancel 返回购物车
  5. 再次点击 Checkout
- **实际结果**：表单为空，需重新填写所有信息
- **预期结果**：应保留之前填写的内容（或至少提供“保存信息”选项）
- **建议修复**：在当前会话中缓存用户填写的数据，或在返回时恢复表单内容

---

## 缺陷跟踪说明

- 以上缺陷均未修复（截至测试日期 2026-03-28）。
